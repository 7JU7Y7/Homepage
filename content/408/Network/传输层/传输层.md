> [!note] 功能与服务
> 实现**进程与进程**间（端到端）的==逻辑通信==
> 
> 实现**复用**和**分用**
> 
> 对==首部和数据==的**差错检测**
> 
> 提供**面向连接可靠**的和**无连接不可靠**的服务

复用：**发送方**的**不同进程**可以使用同一个传输层协议传输数据

分用：**接收方**传输层剥去报文段首部后能将数据**正确交付给目标进程**

![](Pasted%20image%2020250623112109.png)

端口：[^1]唯一标识一个主机内的进程，范围为0-65535（$2^{16}$） ^21c76f

套接字：屏蔽了各个协议的通信细节，提供了tcp/ip协议的抽象，对外提供了一套接口，同过这个接口就可以统一、方便的使用tcp/ip协议的功能。

| 特点 \ 协议    | UDP                    | TCP                          |
| ---------- | ---------------------- | ---------------------------- |
| 面向连接/无连接   | 无连接服务，没有连接状态           | 面向连接的服务                      |
| 可靠数据传输     | 不可靠数据传输                | 可靠数据传输                       |
| 面向应用报文/字节流 | 面向应用报文，报文段长度由应用进程决定    | 面向字节流，报文段长度由窗口和网络拥塞程度决定      |
| 单播、广播、组播   | 支持单播、广播、组播             | 支持单播                         |
| 首部及开销      | 包含源、目的端口、长度（首部+数据）和检验和 | 包含源、目的端口、序列号、确认号、数据偏移、窗口、控制位 |
| 流量控制       | 以最快速度交付，无流量控制          | 滑动窗口实现流量控制                   |
| 拥塞控制       | 以最快速度交付，无拥塞控制          | 慢启动、拥塞避免、快重传，快恢复             |
| 差错检验       |                        |                              |
